# Техническое руководство: PWA-установка приложения «Питание»

Все технические компоненты в проекте **уже настроены и готовы к работе**.  
Для запуска требуется только одно — разместить сайт на хостинге с HTTPS.

---

## 1. Главное требование — HTTPS

Chrome и Safari разрешают установку PWA **только по защищённому протоколу `https://`**.

- Открытие файла напрямую с компьютера (`file:///...`) или по `http://` — **не работает для установки**.
- Рекомендуемые хостинги с бесплатным SSL: **Netlify**, **Vercel**, **GitHub Pages**, **Beget**, **Reg.ru**.

После загрузки на хостинг установка заработает автоматически.

---

## 2. Файлы проекта — не перемещать

При загрузке на хостинг все файлы должны лежать **в одной папке**, рядом с `index.html`:

| Файл | Назначение |
|---|---|
| `manifest.json` | Описание PWA: название, иконки, цвета |
| `sw.js` | Service Worker — кэширование, офлайн-режим |
| `icon-192.png` | Иконка приложения (маленькая) |
| `icon-512.png` | Иконка приложения (большая) |

---

## 3. Логика установки по платформам

В `index.html` встроен полноэкранный экран-блокировщик, который **не имеет кнопки закрытия**.  
Пользователь не может пройти дальше, пока не установит приложение или не откроет сайт в правильном браузере.

### Алгоритм работы:

```
Пользователь открывает сайт
│
├── Уже установлено как PWA (standalone-режим)? → Пускаем сразу ✅
│
├── Android
│   ├── Google Chrome → Ждём 500мс
│   │   ├── Chrome показал промпт установки → Кнопка «Установить» (нативная) ✅
│   │   └── Промпт не пришёл (нет HTTPS) → Инструкция: найди ⊕ в адресной строке
│   ├── Telegram / VK / Instagram (WebView) → Кнопка «Открыть в Chrome» (Intent-переход)
│   └── Другой браузер (Samsung, Opera...) → Кнопка «Открыть в Chrome» (Intent-переход)
│
├── iPhone / iPad (iOS)
│   ├── Safari → Инструкция: нажми «Поделиться» → «На экран Домой»
│   ├── Telegram / VK / Instagram (WebView) → Кнопка «Открыть в Safari» (x-safari-https://)
│   └── Chrome / другой браузер на iOS → Кнопка «Открыть в Safari»
│
└── Десктоп (macOS / Windows)
    ├── Google Chrome → Ждём 500мс
    │   ├── Chrome показал промпт → Кнопка «Установить» (нативная) ✅
    │   └── Промпт не пришёл (нет HTTPS) → Инструкция: найди ⊕ в адресной строке Chrome
    └── Другой браузер (Firefox, Edge, Safari...) → Кнопка «Скопировать ссылку» → открыть в Chrome
```

---

## 4. Как выглядит установка для пользователя

### Android (Google Chrome, HTTPS-сайт)
1. Открывают ссылку в Chrome
2. Появляется полноэкранный экран с кнопкой **«Установить»**
3. Chrome показывает системный диалог — нажимают **«Установить»**
4. Приложение появляется на главном экране и в меню приложений ✅

### iPhone (Safari, HTTPS-сайт)
1. Открывают ссылку в Safari
2. Появляется полноэкранный экран с инструкцией
3. Нажимают **«Поделиться»** (квадрат со стрелкой внизу экрана)
4. Выбирают **«На экран „Домой"»**
5. Нажимают **«Добавить»** ✅

### Telegram / VK / Instagram (Android)
1. Открывают ссылку внутри мессенджера
2. Появляется полноэкранный экран с кнопкой **«Открыть в Chrome»**
3. Нажимают — мгновенный переход в Chrome через системный Intent
4. Далее — стандартная установка через Chrome ✅

### Telegram / VK / Instagram (iPhone)
1. Открывают ссылку внутри мессенджера
2. Появляется полноэкранный экран с кнопкой **«Открыть в Safari»**
3. Нажимают — переход в Safari через схему `x-safari-https://`
4. Далее — стандартная установка через Safari ✅

### Десктоп (Chrome, HTTPS-сайт)
1. Открывают ссылку в Chrome
2. Если PWA-критерии выполнены — кнопка **«Установить»** (нативный промпт)
3. Если промпт не пришёл — инструкция найти иконку **⊕** в адресной строке Chrome и нажать **«Установить»** ✅

### Десктоп (Firefox / Edge / Safari)
1. Открывают ссылку
2. Появляется кнопка **«Скопировать ссылку»**
3. Копируют ссылку и открывают в Chrome
4. Далее — стандартная установка через Chrome ✅

---

## 5. Почему не работает при тестировании локально?

При открытии файла через `file:///...` (прямо с диска) Chrome **не выдаёт промпт установки** — это намеренное ограничение безопасности.

**Для теста** можно использовать локальный HTTPS-сервер:
```bash
# Если установлен Node.js:
npx serve .
# Затем открыть http://localhost:3000 — промпт появится
```

Либо просто загрузить на хостинг и проверить по реальному `https://` адресу.
